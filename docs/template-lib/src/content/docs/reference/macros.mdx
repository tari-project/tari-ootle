title: Macros Reference
description: Documentation for the essential macros provided by the Tari Ootle Template Library.

The Tari Ootle Template Library leverages several powerful Rust macros to simplify common tasks and provide a more ergonomic development experience. These macros handle boilerplate code, abstract away low-level ABI interactions, and streamline event emission and logging.

### `#[template]` Macro

This attribute macro is the most fundamental macro for defining a Tari Ootle Template. It transforms a standard Rust module into a fully-fledged template component that the Tari Ootle validator engine can understand and execute.

-   **Location:** `tari_template_macros` (re-exported by `tari_template_lib`).
-   **Purpose:**
    -   Generates the necessary Application Binary Interface (ABI) for your template.
    -   Exposes public functions within your `impl` block as callable methods of the component.
    -   Ensures your component's state can be persisted on the blockchain.
-   **Usage:** Apply it to a module that contains your component struct and its implementation.
    ```rust
    use tari_template_lib::prelude::*;

    #[template]
    mod MyTemplate {
        pub struct MyComponent {
            value: u64,
        }

        impl MyComponent {
            pub fn new(initial_value: u64) -> Self {
                Self { value: initial_value }
            }

            pub fn get_value(&self) -> u64 {
                self.value
            }
        }
    }
    ```
-   **Details:** Refer to the [Writing Templates](/guides/writing-templates/) guide for a deeper dive into its usage and implications.

### `emit_event!` Macro

Used to emit custom events from within your template. Events are a primary mechanism for Templates to communicate with off-chain applications and services.

-   **Location:** `tari_template_lib::prelude`.
-   **Purpose:** To serialize a Rust struct into an event payload and send it to the Tari Ootle engine, making it observable by external systems.
-   **Usage:**
    ```rust
    use tari_template_lib::prelude::*;

    #[derive(Debug, Event, serde::Serialize, serde::Deserialize)]
    #[event(topic = "MyEventTopic")]
    struct MyEvent {
        msg: String,
        did_something: bool,
    }

  impl MyComponent {
    // A component method:
    pub fn do_something_and_emit(&self) {
        let did_something = self.do_something();

        let event = MyEvent {
            msg: "We might have done something".to_string(),
            did_something,
        };
        emit_event!(event);

        // or emit an event without defining a struct:
        emit_event("MyEventTopic", metadata![
            "msg" => "another msg",
            "did_something" => did_something,
        ]);
    }
}
    ```
-   **Details:** See the [Handling Events](/guides/handling-events/) guide for more information on defining event structures and consuming events.

### Logging Macros (`info!`, `warn!`, `error!`)

These macros are standard Rust logging macros, but within the context of a Tari Ootle Template, their output is directed to the validator node's log files. They are essential for debugging and monitoring the execution of your templates.

-   **Location:** Standard Rust `log` crate (re-exported or integrated).
-   **Purpose:** To output diagnostic messages to the validator's console or log files.
-   **Usage:**
    ```rust
    use tari_template_lib::prelude::*;

    pub fn debug_method(&self) {
        info!("This is an informational message.");
        warn!("This is a warning message, something might be off.");
        error!("This is an error message, something went wrong!");
    }
    ```
-   **Distinction from Events:** Logging macros are for internal debugging and operational monitoring within the validator environment. `emit_event!` is for structured, external communication that off-chain applications can reliably parse and react to.

### Other Macros (Conditional)

Depending on the features enabled in your `Cargo.toml` (e.g., `serde` features, `borsh`), you might encounter other macros like `#[derive(serde::Serialize, serde::Deserialize)]` or `#[derive(borsh::BorshSerialize, borsh::BorshDeserialize)]`. These are standard Rust ecosystem macros used for serialization and deserialization, critical for persistent state and inter-component communication.
